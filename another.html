    프로그래밍 기초

    • 컴퓨터란?
      • 사용자가 입력(input)하면, 값을 처리(processing)해서, 결과를 출력(output)한다.
      • 입력-출력 : 하드웨어-일치하지 않을 수 도 있다.

      번외:다음을 편하게 만드는 게으른 사람의 본성이 만든 작업
      필요한 일을 컴퓨터가 할 수 있도록 특정하게 만드는 작업(코딩, 프로그래밍)


    • 프로그래밍 중에 코딩 과정이 있다.
      • 프로그래밍 : 프로그램을 작성하는 일이라는 뜻. 프로그램을 작성하는 과정에는 설계와 코딩, 에러 수정 등의 단계를 거친다
      • 코딩 : 다른 뜻은 부호화로, 주어진 데이터를 컴퓨터가 이해할 수 있는 형태로 바꾸는 것을 의미한다.


      • 컴퓨테이셔널 씽킹 : 현실 세계의 문제를 분석하여 해결책을 찾는 사고법을 컴퓨테이셔널 씽킹이라하며 이렇게 설계한 해결책을 컴퓨터의 명령어로 작성하는 것

        • 코딩 잘하는 방법
        -검색하여 찾아본다
        - 우리가 식빵에 잼을 바르는 방법이라고 생각하는 것과 컴퓨터가 생각하는 방법은 다르다


      • 번외 : 코딩을 어려워하는 이유
      - 사고하는거를 컴퓨터로의 언어로 번역을 실패/성공

    • 프로그래밍을 잘하는 방법
    • 프로그래밍 언어도 언어
      • 컴파일 언어, 인터프리터(스크립트)언어로 나뉘어진다.
    • 컴파일 언어 - 소스코드를 컴파일 한 후 기계어를 CPU/메모리를 통해 읽어 실행하는 방식으로 동작하는 언어
    • 인터프리터 언어 - 소스코드를 바로 실행하는 컴퓨터 프로그래림, 환경을 말한다.
      • 원시코드를 기계어로 번역하는 컴파일러와 대비된다.
      java, c, c++
        - 컴퓨터는 0, 1 밖에 입력받을 수 없다(기계어)
        - 문자로 작성된 소스코드들을 실행하기 위해서는 반드시 기계어로 바꿔야한다
        - 한번에 다 읽어서 번역하는 것 : 컴파일이라고 부른다
      javascript, R, 파이썬, 루비, PHP - 한줄씩 읽어서 번역하는 것 : 스크립트 언어, 인터프리터 언어
    • 알고리즘, 자료구조
        - 식빵에 잼을 바르는 방법 -> 알고리즘이라고 부른다
        - 10만개를 어떻게 깔아주느냐(관리하느냐) => 자료구조라고 부른다
        - 좋은 프로그랠밍 = 효율적인 알고리즘과 효율적인 자료구조


    • 순서도
    • 프로그램이 수행하는 명령에 대한 순서와 절차를 시각적으로 표현한 도표 순서도를
      그대로 코드로 옮기면 프로그램은 의도한 동작을 그대로 수행한다.

    • REPL(Read, Evaluation, Print, Loop)

      • Read
        • - 입력된 명령어를 읽고
        - 인터프리터가 번역한다 -> 기계어
        - 문자=> 숫자로 변환
      • Evaluation
        • - 실행
      • Print
        • - 실행결과를 돌려준다.
      • Loop
        • - 이 과정을 반복

    URL과 URI의 차이
    • URI(Uniform Resource Identifier)
      resource를 나타내는 모든 종류의 문자열
    • URL(Uniform Resource Locater)
      인터넷에서 웹페이지, 이미지, 비디오 등의 위치
